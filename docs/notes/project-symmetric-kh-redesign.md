# project-symmetric-kh-redesign

> "KH 측정 대칭화 재설계 확정안 — 저장수조 직접 폭기, tank 우선, KCl 스윕=tank 헹굼, 3M KCl 보관 — 구현+Windows 배포 완료(2026-06-14)"

---

**2026-06-12 설계 확정 (사용자 주도), WSL bin/ 구현. 2026-06-14 펌프시간 튜닝 + Windows `measure_kh_once.py` 배포 완료(커밋 9b333e2).** 수렴 판정(`measure_converged`: 45초 간격, 연속 차 <CONV_EPS=**0.005**, 최대 6회, STABLE_SECS=60). 실제 펌프 동작시간은 [project-pump-timings](project-pump-timings.md) 참조(아래 시퀀스의 :5/:2는 설계 당시 placeholder). dkh.dat 형식은 불변(서버 호환), 수렴 횟수는 measure_kh.log 에만 기록.

**문제:** 기존 시퀀스는 참조수가 측정컵에서 30분 폭기(프로브 30분 소크=완전 평형) 후 측정, 수조수는 교체 ~20초 만에 측정 → 전극 응답 지연이 tank에만 실리는 계통 편향. 드리프트(0.01 pH/h)는 ref↔tank 간격이 짧아 무시 가능(~0.002 dKH); 표적은 지연 편향.

**확정 시퀀스 (tank 우선):**
```
[준비]  airoff → ton → m1f:5 (수조수→챔버)
[폭기]  ron 30분 (저장수조 5L + 챔버 동시) — 프로브는 측정컵 KCl에 휴식 → airoff
[tank]  m3b:5 (KCl 배출)
        m2f:2 → m2b:2   ← KCl 스윕 = tank 헹굼 겸용
        m2f:5 → T초 안정화 → 'tank' → m2b:5
[ref]   m4f:2 → m4b:2   ← ref 헹굼 (tank 잔막 제거)
        m4f:5 → T초 안정화 → 'ref' → 'calkh' → m4b:5
[정리]  m1b:5 (KCl 포함 챔버수→본수조 방출·희석) → m3f:5 (KCl 재공급) → airoff
```

**설계 근거:**
- 폭기를 저장수조 자체에서: 새 용기/펌프/펌웨어 불필요, 유일한 물리 작업=참조측 에어스톤을 측정컵→저장수조 이전. 5L 관성으로 ref 평형 안정.
- tank 우선: KCl 스윕이 tank 헹굼을 겸해 단계 절약. 쓸린 KCl은 챔버→m1b→본수조(무한 희석). ref 헹굼 반환수가 KCl-free → 5L 앵커(8.448) 보호.
- 펌웨어는 측정 순서 무관: ref/tank 플래그 독립(ino:618-619), calkh는 둘 다 요구 후 리셋(ino:404-430).
- 헹굼=희석 원리(잔막 0.3mL/채움 50mL ≈ 1/100), 펌프 제트가 교반 역할, 흔들기 불필요. ref 헹굼은 화학적으론 보험이지만 **타이밍 대칭(지연 공통모드 상쇄)을 위해 유지** — tank/ref 블록의 헹굼·채움·안정화 시간을 초 단위로 동일하게.

**보관액: 3M KCl in RODI (1L당 224g ≈ 22% w/v, 시약 직접 제조)**
- 용해도 OK: 포화 ≈4.2M@25°C, 3M은 0°C 근처까지 석출 없음 (이게 3M이 표준인 이유).
- 측정 외 상시 소크 = 컨디셔닝 그 자체 → **별도 재생 루틴 불필요.** 응답 둔화(수렴 횟수 증가로 감지) 시에만 묽은 HCl 세척 고려.
- 중성염이라 스윕 혼입 dKH 무영향. 본수조 K⁺ 유입 하루 ~1g KCl(잔막 0.2mL 가정) → 환수 길면 가끔 K⁺ 테스트.
- 저장조 증발 농축 → RODI 수위 보충.

**스크립트 구현 메모 (다음 세션):**
- 안정화 T = 60~120초, tank/ref 동일.
- 수렴 판정 권장: 'tank'/'ref'를 45초 간격 반복, 연속 차 <0.005이면 진행(최대 6회). 펌웨어가 마지막 값 덮어쓰므로 수정 불필요. 수렴 횟수 로그 = 전극 건강 지표.
- 참조수 5L 증발 농축(1%=+0.08 dKH 상당) → 주기적 수위 보충/calref.
- 에어스톤 이전 직후 첫 사이클은 폭기 1~2시간 수동 연장(5L 첫 평형).
- 에어스톤 이전 직후 첫 사이클은 폭기 수동 연장 권장(5L 첫 평형). 평상시 폭기=20분(AIR_SECS 1200).

**커밋/문서 (2026-06-12, 커밋 9979dfb 푸시 완료):** bin 스크립트 + `docs/system-setup.md` "자동 측정 시퀀스" 섹션 전면 재작성(설계 배경·9단계 흐름·스크립트 상수표·운용 메모) + 문서/SVG의 "KCL 3%"→"3M KCl" 일괄 변경.

**KCl 스윕 정당성 (사용자 질문에 답한 논리, 재논의 방지용):** 스윕한 KCl이 수조물 비이커 경유로 측정수에 일부 되돌아오는 것 맞음(반환 비율≈채움량/비이커수량≈1/3, 헹굼 없으면 ~18mM vs 있으면 ~6mM). 그래도 헹굼이 나은 이유: ①되돌아온 혼입은 **묽고 균일** — 중성염이라 알칼리도 무영향, 이온강도 +2% 수준으로 pH 영향 <0.002. ②헹굼이 막는 진짜 문제는 3M 원액 잔막의 **국소 농도 구배** → 정션 전위 드리프트(수 mV, 1mV≈0.017pH≈0.3dKH) + 수렴 지연. ③ref 블록에 헹굼이 있으므로 tank 헹굼 제거 시 전극 이력 비대칭 재발. ④혼입분은 m1b로 본수조 방출되어 누적 없음.

**Why:** 응답 지연 비대칭 제거가 목표. 사용자 결정 사항(저장수조 폭기, tank 우선, 3M KCl)을 보존해 다음 세션에서 재논의 없이 구현.
